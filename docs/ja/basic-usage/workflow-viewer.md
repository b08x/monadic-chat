# Workflow Viewer

Workflow Viewer は、選択中のアプリの構成をライブ図解します。入力がシステムプロンプトとモデルを経て応答になる流れと、アプリが利用できるツール・エージェント・機能を一望できます。System Settings パネルの **Workflow Viewer** コントロールから開きます。

## 表示モード

| モード | ボタン | 動作 |
|------|--------|----------|
| 非表示 | 目に斜線のアイコン | ビューアを閉じます。 |
| インライン | パネルアイコン | 設定エリア内に図を埋め込み表示します。 |
| フローティング | 外部リンクアイコン | 移動・リサイズ可能なパネルとして開きます。 |

ビューア内のツールバーで、ズームイン/アウト、全体表示、SVG ダウンロード、閉じる、が行えます。

## 図の読み方

ノードは役割ごとに色分けされ、下部の凡例にカテゴリー（Input/Response、Speech、Prompt、Model、Tool、Agent、Feature、Context）が示されます。縦のラインが会話の流れ（入力 → プロンプト → モデル → 応答、音声が有効な場合は音声入出力も）を表し、横のノードがアプリのツール・エージェント・機能・Monadic コンテキストスキーマを表します。

### ツールとスキルグループ

**Tools** ノードには、アプリ固有のツールと共有スキルグループが並びます。グループはクリックで展開・折りたたみできます。

- **🔒（ロック）** — *条件付き*スキルグループ: アプリはこれらのツールに到達できますが、AI が `request_tool` でリクエストした後にのみ提供されます（[ツールグループ](../advanced-topics/tool-groups.md)参照）。
- **🔓（解放済み）** — この会話で既に解放されたグループ。以降のセッションでツールが利用可能です。
- **✓** — このセッションで解放された個々のツール。

会話の途中で AI がスキルを解放すると、図は自動的に更新されます。チャートは常にセッションの実際の能力セットを反映します。解放済みスキルはセッションのエクスポートに保存され、インポート時に復元されます。

### どのアプリがツールを拡張できるか

拡張可能（条件付き）なスキルはアプリのレシピで宣言されるため、アプリごとに異なります:

- tools ブロックの `import_shared_tools :group, visibility: "conditional"` で、そのグループがオンデマンドでリクエスト可能になります。`visibility: "always"` なら最初から含まれます。
- `reachable_skills :group1, :group2`（または厳選された読み取り専用プール `reachable_skills :safe`）は短縮記法です。
- **Knowledge Base 検索**（`library_search`）は、ツール対応アプリに条件付きスキルとして自動追加され、サイドバーの *Use Knowledge Base for retrieval* トグルがオンのときに利用可能になります。

Tools ノードにまったく表示されないスキルグループは、そのアプリでは獲得できません — AI にリクエストを頼んでも成功しません。
